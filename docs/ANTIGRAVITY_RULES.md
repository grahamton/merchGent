# Four Anti-Gravity Rules

These rules are "Always On" for this project.

- **@antigravity Rule 1: One-Layer Only**
  Never propose new layers, services, or abstractions. Solve only at the current level (UI, API, or data) using concrete code diffs.
  Activate with: `@antigravity-layer1`.

- **@antigravity Rule 2: No Scope Creep**
  All suggestions must fit within a single ticket or session. Larger ideas are logged as TODOs without design elaboration. Multi-feature plans are rejected.

- **@antigravity Rule 3: Evidence First**
  Require a working prototype or test before generalizing. After a spike, pause and decide: “Ship, refine, or discard?”

- **@antigravity Rule 4: Guardrail Lead**
  Start every response with:
  `Assumptions: [...]`
  `Out-of-scope: [...]`
  `Risks: [...]`
  Then deliver the trimmed solution.
